## Tutorial
A tutorial should be a single markdown document named after the hmtl/css filename + _tutorial and should be placed in the tutorial folder.

### Structure:
- Brief introduction about the role and function of the document in relation to the website
- Table with three columns:
    - first column: the entire code of the html/css document
    - second column: detailed explanation of the code in English
    - third column: detailed explanation fo the code in Italian
=> create rows according to the document main structure eg: head, body...

The explanation should be detailed with explicit references to the code (eg. this element/attribute does this...). 
Always keep in mind I have no previous knowledge of hmtl/css so use a beginnerfriendly language, but at the same time use the right technical terms.
