## Tutorial
A tutorial should be a single Markdown document named after the HTML/CSS filename + `_tutorial` and placed in the `tutorials` folder.

### Structure:
- Brief introduction about the role and function of the document in relation to the website
- Table with three columns:
    - first column: the entire code of the HTML/CSS document
    - second column: detailed explanation of the code in English
    - third column: detailed explanation of the code in Italian
=> create rows according to the document's main structure (e.g., head, body).

The explanation should be detailed, with explicit references to the code (e.g., "this element/attribute does this").
Assume no prior knowledge of HTML/CSS: use beginner-friendly language while still using correct technical terms.
