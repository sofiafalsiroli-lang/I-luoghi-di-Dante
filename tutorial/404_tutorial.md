# Tutorial: 404.html

## Introduction
`404.html` is the **error page** of the *Dante's Journey* website. Whenever a visitor tries to open a URL that does not exist on the site (for example by mistyping an address or following a broken link), the web server automatically shows this page instead of a blank or cryptic browser error. The page is intentionally very simple: it contains a short message explaining what happened and a link back to the homepage. Like `index.html`, all its styling is written directly inside the file using an internal `<style>` block — no external stylesheet is needed because the page is so small.

| Code | Explanation (English) | Spiegazione (Italiano) |
|------|-----------------------|------------------------|
| `<!DOCTYPE html>` | The very first line of every HTML file. It is a **declaration** (not a tag) that tells the browser to use the modern HTML5 standard when displaying the page. Without it, some browsers might render the page incorrectly. | La primissima riga di ogni file HTML. È una **dichiarazione** (non un tag) che dice al browser di usare lo standard moderno HTML5. Senza di essa, alcuni browser potrebbero visualizzare la pagina in modo errato. |
| `<html lang="en"> … </html>` | The **root element** — the outermost container that wraps the entire page. The `lang="en"` attribute declares that the page content is in English, which helps screen readers pronounce the text correctly and helps search engines understand the language. | L'**elemento radice** — il contenitore più esterno che racchiude l'intera pagina. L'attributo `lang="en"` dichiara che il contenuto della pagina è in inglese, aiutando gli screen reader a pronunciare il testo correttamente e i motori di ricerca a identificare la lingua. |
| `<head> … </head>` | The `<head>` section is **not visible** to the user. It contains metadata and resources the browser needs before rendering anything: the page title, character encoding, viewport settings, favicon, and internal CSS. | La sezione `<head>` **non è visibile** all'utente. Contiene i metadati e le risorse di cui il browser ha bisogno prima di visualizzare qualcosa: il titolo della pagina, la codifica dei caratteri, le impostazioni del viewport, il favicon e il CSS interno. |
| `<meta charset="UTF-8">` | Sets the **character encoding** to UTF-8. This ensures that all characters — including accented letters and special symbols — are displayed correctly instead of appearing as strange symbols. | Imposta la **codifica dei caratteri** su UTF-8. Questo garantisce che tutti i caratteri — comprese le lettere accentate e i simboli speciali — vengano visualizzati correttamente invece di apparire come simboli strani. |
| `<title>404 Not Found</title>` | Sets the **text shown in the browser tab**. Here it reads "404 Not Found", which is the standard HTTP status code for a missing page, so the user immediately knows something went wrong. | Imposta il **testo visualizzato nella scheda del browser**. Qui recita "404 Not Found", il codice di stato HTTP standard per una pagina mancante, in modo che l'utente sappia immediatamente che qualcosa è andato storto. |
| `<meta name="viewport" content="width=device-width, initial-scale=1.0">` | Makes the page **responsive** on mobile devices. `width=device-width` adapts the page width to the screen, and `initial-scale=1.0` sets the default zoom to 100% so the text is not tiny on smartphones. | Rende la pagina **responsiva** sui dispositivi mobili. `width=device-width` adatta la larghezza della pagina allo schermo, e `initial-scale=1.0` imposta lo zoom predefinito al 100% in modo che il testo non sia minuscolo sugli smartphone. |
| `<link rel="icon" type="image/svg+xml" href="data/pictures/favicon.svg">` | Loads the **favicon** — the small icon shown in the browser tab. `rel="icon"` identifies the purpose of the link, `type="image/svg+xml"` specifies the SVG file format, and `href="data/pictures/favicon.svg"` is the file path. Even error pages show the site favicon to maintain brand consistency. | Carica il **favicon** — la piccola icona visualizzata nella scheda del browser. `rel="icon"` identifica lo scopo del link, `type="image/svg+xml"` specifica il formato SVG e `href="data/pictures/favicon.svg"` è il percorso del file. Anche le pagine di errore mostrano il favicon del sito per mantenere la coerenza visiva. |
| `<style> … </style>` | An **internal CSS block** — style rules written directly in the HTML file. The rules here control the background colour, text colour, font, and layout of the error page. | Un **blocco CSS interno** — regole di stile scritte direttamente nel file HTML. Le regole qui controllano il colore di sfondo, il colore del testo, il carattere e il layout della pagina di errore. |
| `body { font-family: Arial, sans-serif; background: #fff; color: #b22222; text-align: center; padding: 5em 1em; }` | Styles the entire visible page. `font-family: Arial, sans-serif` sets a clean font (Arial first, then any sans-serif if Arial is unavailable). `background: #fff` gives a white background. `color: #b22222` makes all text deep red — the same Dante-theme red used throughout the site. `text-align: center` centres every element on the page. `padding: 5em 1em` adds generous spacing: 5em top/bottom and 1em left/right, so the message is not crammed against the edges. | Stilizza l'intera pagina visibile. `font-family: Arial, sans-serif` imposta un carattere pulito. `background: #fff` dà uno sfondo bianco. `color: #b22222` rende tutto il testo rosso scuro — lo stesso rosso del tema dantesco usato in tutto il sito. `text-align: center` centra ogni elemento della pagina. `padding: 5em 1em` aggiunge ampio spazio: 5em sopra/sotto e 1em sinistra/destra, così il messaggio non è compresso ai bordi. |
| `h1 { font-size: 3em; margin-bottom: 0.5em; }` | Styles the main heading. `font-size: 3em` makes it three times the base font size — large and immediately noticeable. `margin-bottom: 0.5em` adds a small gap between the heading and the paragraph below it. | Stilizza il titolo principale. `font-size: 3em` lo rende tre volte più grande della dimensione base del carattere — grande e immediatamente visibile. `margin-bottom: 0.5em` aggiunge un piccolo spazio tra il titolo e il paragrafo sottostante. |
| `p { font-size: 1.3em; }` | Makes all paragraph text slightly larger than the default (1.3× the base size) so it is easy to read. | Rende il testo di tutti i paragrafi leggermente più grande del predefinito (1,3× la dimensione base) per facilitarne la lettura. |
| `a { color: #b22222; text-decoration: underline; }` | Styles all hyperlinks (`<a>` elements) on the page. `color: #b22222` keeps the link in the same red as all other text so it blends with the design. `text-decoration: underline` adds the standard underline to make it clear the text is clickable. | Stilizza tutti i collegamenti ipertestuali (elementi `<a>`) nella pagina. `color: #b22222` mantiene il link nello stesso rosso degli altri testi, coerente con il design. `text-decoration: underline` aggiunge la sottolineatura standard per indicare che il testo è cliccabile. |
| `<body> … </body>` | Contains all the **visible content** of the page — everything the user actually sees. | Contiene tutti i **contenuti visibili** della pagina — tutto ciò che l'utente vede. |
| `<h1>404 – Page Not Found</h1>` | The `<h1>` tag creates the most important **heading** on the page. The text "404 – Page Not Found" is the standard error message: "404" is the HTTP status code meaning the requested resource does not exist, and "Page Not Found" explains it in plain English. | Il tag `<h1>` crea il **titolo** più importante della pagina. Il testo "404 – Page Not Found" è il messaggio di errore standard: "404" è il codice di stato HTTP che indica che la risorsa richiesta non esiste, e "Page Not Found" lo spiega in inglese semplice. |
| `<p>Sorry, the page you are looking for does not exist.</p>` | A `<p>` (paragraph) element containing a friendly apology message. It reassures the visitor that the missing page is a known issue, not a problem with their device or connection. | Un elemento `<p>` (paragrafo) contenente un messaggio di scuse amichevole. Rassicura il visitatore che la pagina mancante è un problema noto, non un problema del suo dispositivo o della connessione. |
| `<p><a href="/">Return to the homepage</a></p>` | A second paragraph containing a **hyperlink** (`<a>` tag). The `href="/"` attribute points to the **root** of the website (the homepage). This gives the user an easy way to navigate back to a working page rather than being stuck on the error page. | Un secondo paragrafo contenente un **collegamento ipertestuale** (tag `<a>`). L'attributo `href="/"` punta alla **radice** del sito (la home page). Questo offre all'utente un modo semplice per tornare a una pagina funzionante invece di restare bloccato sulla pagina di errore. |
